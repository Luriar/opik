# GitHub Preparation Audit

Audit date: 2026-06-22

## Executive Summary

The repository is not ready for public push in its current Git state. The working tree contains about 3.30 GiB across 27,983 files; `.venv` accounts for 705.849 MiB and `.git` for 645.337 MiB. Project-owned files outside those trees total about 2.03 GiB. Git currently tracks 324 files totaling about 905 MiB.

Critical publication findings:

- Two tracked files exceed GitHub's 100 MB per-file limit.
- Three additional untracked runtime archive files exceed 100 MB.
- Oversized blobs already exist in Git history; `.gitignore` cannot remove them from history.
- Generated CSV, XLSX, Parquet, models, daily results, and archives are mixed with source-controlled material.
- The existing `.gitignore` covers virtual environments, caches, outputs, logs, and Parquet, but not the requested data trees, CSV, XLSX, or archive rules.
- No obvious committed secret or credential file was found, and no credential assignment pattern was detected outside generated data/report trees.
- `LICENSE`, `SECURITY.md`, `CONTRIBUTING.md`, and `CODE_OF_CONDUCT.md` are absent.
- README and several documents display apparent encoding corruption in the current console and should be UTF-8 reviewed before publication.
- `configs/daily_update.yaml` points to ignored local datasets. A public sample-data/bootstrap explanation is needed so a fresh clone fails clearly or runs in sample mode.

No files were deleted, moved, or ignored during this audit.

## Classification Key

1. **GitHub Required**: source, configuration, tests, or core documentation needed to understand and reproduce the software.
2. **GitHub Optional**: useful research, examples, prompts, or curated human-readable evidence.
3. **Generated Runtime Data**: reproducible pipeline inputs/outputs generated during operation.
4. **Archive Only**: historical snapshots or legacy artifacts that should live outside normal Git history.
5. **Should Be Ignored**: environment, cache, log, binary, or generated export content unsuitable for Git.

Sizes and counts are recursive. `.git` and `.venv` internals are grouped because their 3,314 child directories inherit the same recommendation. All 106 project-owned directories are listed individually.
